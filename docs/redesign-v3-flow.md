## Часть 1. Философия каждого элемента

Чтобы решить, что оставить/упростить/убрать — надо ответить «зачем оно вообще». Прохожу по всем элементам и ставлю вердикт.

### Видение (из README)

> Second brain · growth tracker · memoir.
> Три слоя: **Now** (что делаю сейчас), **Self** (кто я сейчас), **Past** (что было).

Это — единственная истина. Всё, что не вписывается в эти три слоя, — кандидат на убийство или переосмысление.

|---|---|---|---|

### Главные выводы

**A. У продукта 3 слоя — должно быть 3 таба.** Сейчас 10 пунктов навигации, пользователь теряется. Сводим к **Сейчас / Я / Лента**.

**B. ИИ-фичи (Coach, Roadmap, Ассистент) — это утилиты, не место.** Утилиты живут в топбаре или внутри карточек, не как отдельные «места» в навигации.

**C. Метрики прогресса дублируются.** Уровень и эпоха считаются из одного XP — это путает. Эпоха становится визуальной вехой раз в 5 уровней (косметика веток), а главное число — это уровень с прогрессом до следующего.

**D. Награды отсутствуют.** Пользователь копит XP, но не получает за это ничего, кроме цифры. Добавляем **звания** (rank names) — одно на глобальный уровень («Странник» → «Практик» → «Мастер»…), одно на эпоху оси («Зерно» → «Росток» → «Древо»…). Это бесплатная награда, делает прогресс осмысленным.

## Часть 2. Новая навигация

### Мобайл

3 таба в нижней капсуле:

```
┌──────────────┬──────────────┬──────────────┐
│   Сейчас     │      Я       │    Лента     │
│ (что делать) │ (как я расту)│ (что было)   │
└──────────────┴──────────────┴──────────────┘
```

Сверху на каждом экране — общая панель действий (иконки): `[Coach] [План] [Pomodoro] [Настройки]`.

Плюс плавающая «пятиугольная» кнопка добавления записи — как сейчас.

### Десктоп

Сайдбар слева — те же 3 основных пункта вверху + блок «вспомогательное» внизу:
`Задачи`, `Граф знаний`, `Календарь`, `Coach`, `План`, `Pomodoro`, `Настройки`.

Ассистент удаляется полностью.

### Точки входа во вторичные экраны

- **Задачи (TasksScreen)** — push из «Сейчас» (через карточку «Сегодня» → «Все задачи →»).
- **Граф знаний** — push из «Я» (кнопка-карточка «Граф знаний →»). Также из сайдбара десктопа.
- **Календарь** — push из «Лента» (кнопка-карточка «Календарь →»). Также из сайдбара десктопа.
- **Coach / Roadmap / Pomodoro / Настройки** — топбар.

## Часть 3. Новая модель XP / уровней / эпох

### До

| | Откуда | Где |
|---|---|---|
| Глобальный уровень | Σ XP всех осей | self_screen header |
| Уровень оси | Σ XP оси | axis_detail badge + self_screen axis row |
| Эпоха оси | Σ XP оси / 500 | axis_detail + self_screen axis row |
| Pentagon score | 30-дн XP с затуханием | пятиугольник |

Уровень и эпоха — про одно и то же, выражены разными формулами.

### После

| | Откуда | Где |
|---|---|---|
| **Глобальный уровень** + звание | Σ XP всех осей | self_screen header, профиль |
| **Уровень оси** + прогресс до следующего | Σ XP оси | axis_detail badge + self_screen axis row |
| **Эпоха оси** + звание (косметика) | `1 + (axisLevel - 1) ~/ 5` | визуально: цвет/нимб ветки + название эпохи |
| **Pentagon score** | 30-дн XP с затуханием | пятиугольник |

`epochFromXp` и `xpToNextEpoch` помечаются deprecated и заменяются на `epochForLevel(int level)`.

### Звания (rewards)

**Глобальные уровни (по пятёркам):**

| Уровень | Звание |
|---|---|
| 1-4 | Новичок |
| 5-9 | Странник |
| 10-14 | Искатель |
| 15-19 | Практик |
| 20-29 | Мастер |
| 30-49 | Архитектор |
| 50+ | Хранитель |

**Эпохи оси (раз в 5 уровней этой оси):**

| Эпоха | Звание | Визуал |
|---|---|---|
| Э1 (Lv 1-5) | Зерно | ветка без декора |
| Э2 (Lv 6-10) | Росток | нимб тоньше |
| Э3 (Lv 11-15) | Побег | нимб ярче |
| Э4 (Lv 16-20) | Ветвь | двойной нимб |
| Э5 (Lv 21-25) | Крона | тройной нимб |
| Э6+ | Древо / Лес… | каждый шаг — оттенок ярче |

В UI выводим эпоху как название («Э3 Побег»), а не голую цифру.
